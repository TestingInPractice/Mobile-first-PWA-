# AC-005: Workflow Integration Acceptance Criteria

- [x] **AC-1**: Чат принимает команду `/decompose "описание задачи"`
- [x] **AC-2**: Система генерирует 2-4 подзадачи из описания
- [x] **AC-3**: Для каждой подзадачи создаётся GitHub Issue
- [x] **AC-4**: Issues получают label `sub-agent` и `phase`
- [x] **AC-5**: После создания issues обновляется phases.json (статус → in-progress)
- [x] **AC-6**: Дашборд отображает обновлённый статус после перезагрузки
- [x] **AC-7**: Команда `/status` показывает текущее состояние всех фаз
