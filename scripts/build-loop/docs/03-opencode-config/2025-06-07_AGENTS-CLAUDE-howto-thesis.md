# AGENTS.md и CLAUDE.md: как готовить? — структурированные тезисы


## 1. О чём видео

Гайд по написанию AGENTS.md / CLAUDE.md для кодинг-агентов. Разбираются различия между AGENTS.md, CLAUDE.md, GEMINI.md, Cursor rules, структура и оптимальное содержимое.

---

## 2. Поддержка AGENTS.md разными агентами

Каждый агент обрабатывает AGENTS.md по-своему:

**Codex (OpenAI):**
- Ищет `~/.codex/AGENTS.md` (глобальный)
- Ищет `~/.codex/OVERRIDE.md` (переопределение)
- В проекте: корневой `AGENTS.md`

**OpenCode:**
- Ищет `.opencode/rules/` (директория правил)
- Поддерживает `AGENTS.md` в корне проекта

**Cursor:**
- `.cursor/rules/` — директория с правилами
- Поддерживает AGENTS.md

**Gemini Code Assist:**
- `GEMINI.md` в корне проекта

---

## 3. Оптимальная структура AGENTS.md

### Что должно быть:
- **О чём проект** — короткое описание
- **Workflow** — как работаешь с агентом
- **Базовые принципы** — code style, conventions
- **Информация, полезная при каждой операции**

### Чего НЕ должно быть:
- ❌ Команды запуска тестов
- ❌ Инструкции по оформлению PR
- ❌ Полная структура проекта
- ❌ Конвеншены именования

**Принцип:** файл прикрепляется к каждому запросу. Должна быть только та информация, которая полезна агенту **при выполнении каждой операции**.

---

## 4. Глобальные vs локальные правила

- **Глобальный AGENTS.md** — поведение агента во всех проектах
- **Локальный AGENTS.md** — специфика конкретного проекта
- **OVERRIDE.md** (Codex) — полное переопределение глобальных правил для проекта

---

## 5. Пример плохого AGENTS.md

Автор показывает пример из документации, где в AGENTS.md пишут:
- Как запускать тесты
- Как оформлять PR
- Полную структуру директорий

**Проблема:** при запросе "добавить миграцию" агенту не нужно знать, как тестировать фронтенд.

## 7. Ключевые инсайты

- AGENTS.md — это **проектный контракт**, а не документация
- Каждый лишний токен в AGENTS.md — потеря контекста для реальной задачи
- Для долгих инструкций лучше использовать `instructions/` директорию (поддерживается OpenCode)
- OVERRIDE.md в Codex — мощный механизм для проектов со специфическими требованиями
- Разные агенты имеют разный парсинг — читай документацию своего инструмента
