# Как кодинг-агенты используют инструменты, память и контекст

**Источник:** https://habr.com/ru/articles/1021168/
**Оригинал:** Sebastian Raschka, перевод Дмитрия Антипова

---

Разбор архитектуры кодинг-агентов и agentic harness на примере Mini Coding Agent. 6 ключевых компонентов:

1. **Живой контекст репозитория** — сбор стабильных фактов (ветка git, README, AGENTS.md, структура) перед каждым вызовом
2. **Структура промпта / кэширование** — стабильный префикс кэшируется; меняется только история + запрос
3. **Доступ к инструментам** — именованные инструменты с валидацией, sandbox, подтверждениями
4. **Борьба с раздуванием контекста** — обрезка, дедупликация, суммаризация
5. **Структурированная память сессии** — транскрипт + рабочая память
6. **Делегирование и субагенты** — read-only, ограниченная глубина рекурсии

**Главный вывод:** качество харнесса часто важнее качества модели. Именно поэтому одна и та же модель в CLI-агенте (Claude Code, Codex) ощущается мощнее, чем в чате.
