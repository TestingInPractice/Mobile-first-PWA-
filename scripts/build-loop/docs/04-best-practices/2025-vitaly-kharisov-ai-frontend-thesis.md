# «Я 99.99% кода пишу нейронкой» — Виталий Харисов (Яндекс, БЭМ) — тезисы


### Ключевые тезисы

- **99.99% кода пишет нейронкой** — утверждает Харисов. Исключение: только скрипты, критичные к безопасности
- **Open Source vs работа в Яндексе:** в OS — X10 разработчик (один человек = full-stack + менеджер + дизайнер), на работе — X2-X4 (200-400% ускорение)
- **Разница:** в OS можно делать проект в своём темпе, на работе нужна day-by-day отдача и специализация
- **LLM как опытный разработчик:** Харисов общается с нейронкой как с senior-инженером, применяя свой architectural judgment. «Я пишу задачи для кодировщика»
- **Пример:** написал Android-библиотеку компонентов, не зная ни Android, ни Kotlin — всё через диалог с нейронкой. Сначала Compose, потом LLM сама предложила перейти на Java + Kotlin sugar + Compose wrapper для совместимости
- **Пример:** статический сайтогенератор — начал с MkDocs через LLM, перешел на Eleventy, потом полностью переписал под себя. Итог: 800 тестов, минимальный валидный HTML, gzip + brotli на сборке
- **LLM acceleration:** «Раньше — читай доку, пиши код, пиши долбаные тесты. Сейчас — make me everything, и в это время пьёшь чай»
- **Почему нейронки — кайф:** позволили делать проекты, за которые раньше никто бы не взялся (год работы → 1 месяц). Сместили фокус с «как писать» на «что получится»
- **Почему разработчики отказываются:** или не умеют готовить промты, или их домен недостаточно представлен в обучающих данных. Фронтендерам, отказывающимся от AI — «конные фронтендеры, которые будут выступать в цирке»
- **Собеседования:** спрашивать надо не про промты, а про умение решать задачи и «что ты за человек». Алгоритмические задачи на доске — не про код, а про процесс мышления
- **Будущее:** возможен подход, где код — чёрный ящик. PM говорит «переписать», агент делает. Программист может стать не нужен, останутся только продакт-менеджеры
- **Пожелание:** «дешёвых токенов и быстрых, чтобы всё отлетало сразу, и поменьше санкций»

### Контекст-инжиниринг (от Харисова)

- LLM использует как архитектурного партнёра: обсуждает стек, ревьюит код, советуется
- Не читает сгенерированный код целиком — смотрит API, демо, визуальный результат
- AGENTS.md-подход: написал правила про BMS-структуру (скриншоты рядом с кодом), LLM сначала нарушает, потом переделывает по правилам
- Каждая написанная компонента становится reference для следующих («посмотри в радио, сделай кнопку»)
